# 整场直播切片 · 精修 prompt

你要把一份「张雪峰整场直播」按**话题**切成若干份独立片段，并为每份起标题、判内容类型。

## 输入

- `original_text`：整场原文（未过滤）
- `noise_segments`：启发式识别的噪声段 offset 列表（带货/闲聊/主持串场），仅供参考
- `candidates`：规则粗切的候选段 offset + 切分依据，仅供参考（边界可能不准）

## 任务

**输出最终切片方案**。以候选段为起点，但你有权：

1. **调整边界**：让每段起止落在自然话题边界（一个连麦结束 / 一个专题收尾）
2. **合并/拆分**：候选段过碎就合并；一个候选段里混两话题就拆开
3. **丢弃**：候选段大部分是带货/闲聊 → 不要它
4. **打标题**（≤20 字，能概括该段核心问题或主题）
5. **判 content_type_hint**：`对话` / `独白` / `专题` / `其他`
   - 对话：家长连麦 Q&A
   - 独白：张老师单人讲解某个经验/观点
   - 专题：结构化讲某个系统性话题（常有"第一/第二/第三"）
   - 其他：介于之间或难以判定

## 边界规则

- 单段最短 1500 字，最长 8000 字（超了再拆）
- `start` / `end` 必须是**原文字符 offset**（从 0 计），从 original_text 里能精确切出来
- 段间允许有 gap（带货段不进任何片段），但不能重叠
- 一份整场产出 4-12 个片段比较常见；太多说明切碎了，太少说明粘糊了

## 过滤

凡是符合以下任一，**不要**作为独立段：
- 开场/结尾的带货口播（苞米、小黄车、福袋、下单、下播、感谢礼物……）
- 主持串场（"直播间交给老师"之类）
- 纯闲聊（昨天吃啥、狗、堵车……）
- 长度 < 1500 字且话题不独立

## 输出格式

**只输出一个 ```json 代码块**，结构如下：

```json
{
  "parent_bv": "BV1CF2UYcEFw",
  "segments": [
    {
      "title": "口腔医学低分保底",
      "start": 4200,
      "end": 7830,
      "content_type_hint": "对话",
      "rationale": "家长问孩子 480 分口腔医学能不能上"
    },
    {
      "title": "组干净与专业组风险",
      "start": 8100,
      "end": 13600,
      "content_type_hint": "独白",
      "rationale": "张老师单人讲院校专业组不干净的风险"
    }
  ],
  "dropped": [
    {"start": 0, "end": 1200, "reason": "开场带货 + 闲聊"}
  ]
}
```

`rationale` 一句话写清你选这个边界/类型的理由，便于人工抽查。

## 强约束

- 不要输出任何 JSON 以外的文字
- offset 必须在 [0, len(original_text)] 范围内
- 段之间允许 gap 但不能重叠，且 start < end
- `title` 不含 `/`、`\`、空格、`*`、`?`、`"`、`<`、`>`、`|`、`:`（会用于文件名）
